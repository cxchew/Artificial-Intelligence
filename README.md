## 📁 Repository Contents (Project Phases)

Below is an overview of the progressive project phases contained in this repository. Navigate to their respective folders to view the full design reports, logical proofs, state trees, and scenario walk-throughs.

| Project Component | Technical Topic | Core Focus & Artifacts |
| :--- | :--- | :--- |
| 📊 **[Phase 1: Project Proposal]** | Problem Formulation & Design Thinking | Explored the agricultural theme by framing structural climate risks. Built user-centric **Empathy Maps** and **User-Need-Insight Matrices** to isolate explicit agricultural pain points. |
| 🔏 **[Phase 2: Knowledge Representation]** | Declarative Rules & Predicate Logic | Transformed messy environmental anomalies into computer-friendly rules using **First-Order Predicate Logic (FOPL)** quantifiers ($\forall, \exists$) and deterministic **"If-Then" Production Rules**. |
| 🌳 **[Phase 3: State Space Search]** | Pathfinding & Decision Optimisation | Formulated sequential farming operations into searchable states. Modeled planting phase graphs and evaluated **path costs** within corrective monitoring loops. |
| 🤖 **[Phase 4: Intelligent Agent Design]** | PEAS Framework & Closed-Loop Control | Abstracted the entire software system into a formal autonomous agent layout utilizing the **PEAS Matrix** (Performance, Environment, Actuators, Sensors) with closed feedback loops. |
| 🚨 **[Phase 5: Proof of Concept]** | Interface Validation & Live Scenarios | Developed high-fidelity simulated dashboard interfaces to prove how backend logic rules actively translate into clear, actionable advice across **Critical Flood Risk** and **Harvest Optimisation** screens. |

## 💡 Overall Subject Reflection

Completing this course in Artificial Intelligence has fundamentally redefined my approach to system engineering. Moving past standard procedural programming, I have learned that building intelligent software requires transforming chaotic human environments into formal mathematical boundaries where machines can reason, adapt, and make optimized choices.

* **Human-Centric Problem Formulation:** This curriculum taught me that a powerful AI model is only as effective as its initial problem definition. Starting our project with comprehensive design thinking frameworks proved that engineering an intelligent system always begins with deeply understanding real-world human risks and mapping user anxieties into clean technical constraints.
* **The Rigor of Symbolic Machine Reasoning:** Developing the knowledge representation and state space search layers showed me how to turn abstract judgment call parameters into strict logic constraints. Learning to calculate path costs across maintenance nodes and writing formal predicate rules using logical connectors proved that AI systems must maintain open, explainable reasoning paths to remain reliable.
* **Autonomous Feedback and Presentation:** Designing the closed-loop agent architecture and validating it through high-fidelity simulation interfaces demonstrated how an AI system handles unexpected data drifts. I learned how an effective system brain balances short-term emergencies with long-term gains, automatically abstracting complex backend logic payloads into simple, high-visibility dashboard views that guide safe human decisions.
